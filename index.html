Minimal Coding

Very minimal coding is required to set it up. You can be up and running within minutes of installation and even without setting up options prettyValidate is ready to go right out of the box.

HTML5 + CSS3

prettyValidate uses the latest practices in HTML5 so that you can be sure that your form will stay HTML5 compliant, but incase you're not using HTML5 prettyValidate will still work the same.

Configurable

prettyValidate is customizable in any way you can think of. From how the form is submitted, to what happens before or after the fields are validated, to callbacks from any errors it encounters.

Usage

Requirements

prettyValidate requires the use of jQuery v2.2 and up as well as Bootstrap v3.3 and up. You can get each of prettyValidates dependancies by following the links below:

  • jQuery v2.2.x (latest)
  • Bootstrap v3.3.x (latest)
  • colourBrightness.js (if you use the colorpicker)

If you would like to contribute your ideas to make prettyValidate more compatible with older versions of its dependancies, feel free to contact me in regards to becoming a contributor.

Getting Started

Copy & paste the stylesheet <link> into your <head> right before your "style.css" file.

Add the JavaScript plugin after jQuery has been called, right before the closing <body> tag.

.prettyValidate()

To use, simply call the .prettyValidate() function:

.prettyValidate(options)

The following are prettyValidate's options and their defaults:

Options

prettyValidate's options will grow with development as there is plenty of room for improvement and area's that could become changeable with options. If you have an idea for an option, please contact me on Github.

Name Type Default Accepted Values Description
ajax boolean false true, false Submit the form via Ajax.
url string false URL If set, will ignore the forms action="" attribute and will send the Ajax request to this URL. Requires ajax: true
redirect boolean or string false true, or URL If set to true, will redirect to the forms action="" attribute; otherwise will redirect to set URL. Requires ajax: true.
reset boolean true true, false If true, the form will reset once it's successful.
shake boolean true true, false Invalid inputs will shake on submit.
valid function null function(){} Fired once the form is returned valid.
invalid function null function(){} Fired if an invalid field is found.
success function null function(data){} Fired if the Ajax was successful. Requires ajax: true.
error function null function(error){} Fired if an error occurs during Ajax. Requires ajax: true.
complete function null function(data){} Fired as the loading overlay fades out. Requires ajax: true.
beforeSend function null function(){} Fired before the form is sent through validation.
debug boolean false true, false Will output messages in your browsers console about what's going on in the function.

Formatting

Most Bootstrap Form formatting will work prettyValidate inputs. The only exception is that all fields must be wrapped inside of a <fieldset> tag in order for prettyValidate to work.

Text Input

Fields left with type="text" will only allow A - Z in uppercase & lowercase, as well as space's. The input will return invalid if user tries to input any numbers or special charachters.

Email Address

Fields with type="email" will watch for properly formatted email addresses such as email@address.com and if the input does not match the correct formatting, will return invalid.

Phone Number w/ extension

Fields with type="phone" will return true if the phone number is formatted with either 5555555555, 555-555-5555, 555 555 5555, or (555) 555 5555.

If you wish to append a phone number extension field, add a data-ext="true" attribute to the <input> tag.
The extension input will accept only numbers, 5 characters long.

Color Picker

Fields with type="color" will automatically fire the Bootstrap Colorpicker plugin, allowing users to select their color from the popup. If the user types in their own color, validation will make sure that it follows the HEX #FFFFFF pattern, or else will return invalid.

Bootstrap Colorpicker is written by Stefan Petre and the version used was modified by Javier Aguilar.

Date Picker

Fields with type="date" will automatically fire the Bootstrap Datepicker plugin, giving users a visual datepicker. If the user types in the date manually, validation will make sure that it follows the MM/DD/YYYY pattern, or else will return invalid.

Bootstrap Datepicker is written by Stefan Petre and the version used was modified by Andrew Rowls.

Range Field

Input type="range" will be displayed to the user as a pretty / modern looking slider bar, with the option to display the value above it. The value can be displayed by adding a placeholder attribute to the inputs tag, such as placeholder="Range".

The range slider is customizable by adding the min or max attributes. The default value can be chosen by adding the value attribute, and javascript will automatically update value shown above it.

Textarea

<textarea>'s in prettyValidate allow all characters as long as it doesn't detect a HTML string formed. Like any other textarea, the height can be adjusted using rows="".

Multiple Inputs

If you'd like to display multiple inputs inline simply add the class input-group to the parent <fieldset> tag. As of now, prettyValidate only supports 2 inputs within the same fieldset, but eventually will allow for up to 4.
